आपके प्रोजेक्ट के लिए पञ्चमाक्षर नियमों का पूर्णतः पालन करते हुए एक विस्तृत और व्यावसायिक **README.md** फ़ाइल तैयार की गई है। इसमें इन्स्टॉलेशन से लेकर लोकल होस्टिंग (Ngrok) और हगिंग फ़ेस (Hugging Face) पर डिप्लॉयमेन्ट तक की पूरी प्रक्रिया स्पष्ट रूप से समझाई गई है।

आप नीचे दिए गए कोड ब्लॉक की सामग्रियों को कॉपी करके अपने प्रोजेक्ट फ़ोल्डर के अन्दर **`README.md`** नाम की नई फ़ाइल में सहेज सकते हैं:

```markdown
# 🎙️ Gemini Verbatim Transcription & Formatting Engine

यह एक अत्याधुनिक, तीव्र और सुदृढ़ **Streamlit** एप्लिकेशन है, जो **Google Gemini API** (जैसे `gemini-3.5-flash`) का उपयोग करके बड़े ऑडियो और अव्यवस्थित टेक्स्ट को अक्षरशः (Word-to-Word Verbatim) ट्रांसक्रिप्ट और फ़ॉर्मेट करता है। यह टूल इन-मेमोरी बाइनरी स्ट्रीमिंग (In-memory binary streaming) और स्वचालित बैकअप प्रणालियों से लैस है।

---

## 🛠️ मुख्य विशेषताएँ (Core Features)

- **डायनेमिक मॉडल चयन (Dynamic Model Selection):** मन्च पर रनटाइम के दौरान ड्रॉप-डाउन से जेमिनी मॉडल वर्ज़न (`Gemini 3.5 Flash Default`, `Gemini 3.0 Flash`, `Gemini 2.5 Flash`, `Gemini Latest Flash`) बदलने की पूर्ण सुविधा।
- **इन-मेमोरी ऑडियो चंकिंग (In-Memory Audio Chunking):** बिना किसी टेम्पररी फ़ाइल के, `pydub` और `FFmpeg` के माध्यम से बड़े ऑडियो को 2 से 10 मिनट के सुरक्षित खण्डों में विभाजित करना, जिससे बेस64 पेलोड का आकार कभी भी 20 MB से ऊपर नहीं जाता।
- **सुरक्षित पब्लिक/प्राइवेट मोड:** यदि एनवायरनमेंट में `GEMINI_API_KEY` अनुपलब्ध हो, तो ऐप स्वचालित रूप से पब्लिक मोड में रनटाइम पर यूजर से की (Key) स्वीकार करता है।
- **डेटा हानि सुरक्षा (Anti-Data Loss Backup):** कनेक्शन टूटने या सत्र निष्क्रिय (Idle) होने पर भी डेटा नष्ट न हो, इसलिए यह प्रत्येक सफल प्रोसेसिंग के बाद `transcripts_backup/` निर्देशिका में स्वचालित रूप से स्थानीय बैकअप सहेजता है।
- **प्रॉम्प्ट प्रबन्धन (Prompt Directory Management):** हिंदी और अंग्रेज़ी के लिए पृथक बाह्य नियमों (`prompts/` निर्देशिका) को गतिशील रूप से लोड करने की क्षमता।

## 🚀 स्थानीय संस्थापन (Local Installation)

### 1. पूर्वापेक्षाएँ (Prerequisites)

आपके सिस्टम पर **Python 3.12+** या **Python 3.14+** और **FFmpeg** स्थापित होना अनिवार्य है।

* **Mac (Homebrew):** `brew install ffmpeg`
* **Windows:** [FFmpeg Official Website](https://ffmpeg.org/download.html) से डाउनलोड कर System PATH में जोड़ें।

### 2. रिपॉजिटरी सेटअप और पैकेजेस

अपने टर्मिनल में निम्नलिखित कमांड्स चलाएँ:

```bash
# प्रोजेक्ट फ़ोल्डर में जाएँ
cd C:\LearnGita\JV_Transcription

# वर्चुअल एनवायरनमेंट सक्रिय करें (Windows)
.venv\Scripts\activate

# आवश्यक निर्भरताएँ स्थापित करें
pip install streamlit google-genai pydub python-dotenv audioop-lts

```

*टिप्पणी: Python 3.14+ पर `pydub` की आन्तरिक निर्भरता बनाए रखने के लिए `audioop-lts` का उपयोग किया गया है।*

### 3. सीक्रेट्स कॉन्फ़िगरेशन

प्रोजेक्ट के मूल फ़ोल्डर में एक **`.env`** फ़ाइल बनाएँ और अपनी एपीआई की दर्ज करें:

```text
GEMINI_API_KEY="आपकी_वास्तविक_जेमिनी_api_की"

```

### स्थानीय रूप से चलाने के लिए:

```bash
streamlit run tramscrobeV1.py

```

### Ngrok टनल के माध्यम से ऑनलाइन साझा करने के लिए:

बिना गिटहब (GitHub) पर कोड डाले इंटरनेट पर सुरक्षित साझा करने के लिए, फ़ोल्डर में उपलब्ध **`run_ngrok.bat`** फ़ाइल पर डबल-क्लिक करें। यह आपके स्थानीय पोर्ट `8501` को लाइव वैश्विक यूआरएल (जैसे `https://xxxx.ngrok-free.app`) पर मैप कर देगा।

## 🤗 Hugging Face Spaces पर डिप्लॉयमेन्ट

यदि आप इस ऐप को चौबीसों घण्टे बिना अपना कंप्यूटर चालू रखे ऑनलाइन चलाना चाहते हैं, तो Hugging Face पर ये दो फ़ाइलें अवश्य बनाएँ:

1. **`requirements.txt`**:
```text
google-genai
pydub
python-dotenv
audioop-lts

```


2. **`packages.txt`** (FFmpeg के संस्थापन के लिए अत्यंत आवश्यक):
```text
ffmpeg

```



*सुरक्षा टिप: Hugging Face की **Settings -> Variables and secrets** में जाकर `GEMINI_API_KEY` को एक Secret के रूप में सुरक्षित सहेजें।*
